# Описание проекта
SyncTask — это многопользовательская клиент-серверная система управления задачами с графическим интерфейсом. Реализована на чистом Python с использованием TCP-сокетов для сетевого взаимодействия и Tkinter для создания пользовательского интерфейса.

## Технологический стек
Python 3.7+

## Стандартные библиотеки Python:

`socket` — сетевое взаимодействие

`threading` — многопоточная обработка подключений

`tkinter` — графический интерфейс

`json` — сериализация и десериализация данных

`uuid` — генерация уникальных идентификаторов

`re` — регулярные выражения для валидации

`datetime` — работа с датами и временем

# Ключевые возможности
## Система аутентификации и ролей
- Многопользовательская система с предустановленными аккаунтами

- Ролевая модель: администратор и обычные пользователи

- Авторизация по логину и паролю через TCP-сокет соединение

## Управление задачами
- Полный набор CRUD-операций для задач

- Автоматическое обновление списка задач каждые 5 секунд

- Уникальные идентификаторы задач на основе UUID

- Валидация дат с проверкой корректности формата и логики

## Разграничение прав доступа
### Для администратора (admin):
- Просмотр всех задач в системе

- Назначение задач любым пользователям

- Удаление любых задач

- Доступ к списку всех зарегистрированных пользователей

### Для обычных пользователей:
- Просмотр только назначенных им задач

- Изменение статуса своих задач:

  - Не начато

  - В работе

  - Выполнено

## Архитектура проекта
### Серверная часть (Backend.py)
- TCP-сервер на порту 12345

- Многопоточная архитектура для обработки клиентов

- Хранение данных в файле tasks.txt

- Обработка команд через текстовый протокол


### Клиентская часть (Frontend.py)
- Графический интерфейс на базе Tkinter

- Адаптивный интерфейс, меняющийся в зависимости от роли пользователя

- TreeView-компонент для табличного отображения задач

- Валидация вводимых данных на стороне клиента

## Установка и запуск
### Предварительные требования
- Python 3.7 или выше

## Инструкция по запуску
1. Клонирование репозитория

```bash
git clone https://github.com/ваш-username/synctask.git
cd synctask
```
2. Запуск сервера

```bash
python Backend.py
```
Сервер будет запущен на localhost:12345


3. Запуск клиента

```bash
python Frontend.py
```
Запустите столько экземпляров клиента, сколько нужно пользователей.

### Тестовые учетные записи
```Логин	Пароль	Роль

user1	user1	Обычный пользователь
user2	user2	Обычный пользователь
user3	user3	Обычный пользователь
user4	user4	Обычный пользователь

```
### Формат хранения данных
Задачи хранятся в файле tasks.txt в CSV-формате с шестью полями:

```text
UUID,ДатаСоздания,Ответственный,Дедлайн,ОписаниеЗадачи,Статус
```
Пример записи:

```text
a1b2c3d4-e5f6-7890-abcd-ef1234567890,2023-12-01,user1,2023-12-10,Подготовить ежемесячный отчет,В работе
```
### Протокол взаимодействия
Клиент и сервер обмениваются текстовыми командами:

### Команды от клиента к серверу
`login,password` — авторизация пользователя

`tasks` — запрос списка задач

`list_users_admin` — запрос списка пользователей (только для администратора)

`change:UUID:НОВЫЙ_СТАТУС` — изменение статуса задачи

`append,дата,ответственный,дедлайн,описание,статус` — добавление новой задачи

`delete:UUID` — удаление задачи

### Ответы сервера
`YES` — успешная авторизация

`NO` — неуспешная авторизация

JSON-массив задач — в ответ на команду tasks

CSV-список пользователей — в ответ на команду list_users_admin

### Особенности реализации
1. Минималистичность — используются только стандартные библиотеки Python

2. Потокобезопасность — каждый клиент обрабатывается в отдельном потоке

3. Валидация данных — двойная проверка данных на клиенте и сервере

4. Устойчивость к ошибкам — корректная обработка разрывов соединения

5. Логирование — подробное логирование всех операций на стороне сервера

## Пример использования
1. Запустите серверную часть:

```bash
python Backend.py
```
2. Запустите первый клиент с учетной записью администратора:

- Логин: admin

- Пароль: password123

3. Создайте несколько задач для разных пользователей через интерфейс администратора.

4. Запустите второй клиент с учетной записью обычного пользователя:

- Логин: user1

- Пароль: user1

5. Наблюдайте, как пользователь видит только свои задачи и может изменять их статус.

6. В клиенте администратора наблюдайте обновления статусов задач в реальном времени.
